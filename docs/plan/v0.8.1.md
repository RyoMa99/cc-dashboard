# v0.8.1: リポジトリフィルタ選択時に Cost by Repository セクションを非表示にする

## Context

GitHub Issue #28。リポジトリフィルタで特定リポジトリを選択すると、「Cost by Repository」セクションが全リポジトリのコスト内訳を表示し続ける。1つのリポジトリしか表示されないため内訳としての意味が薄く、フィルタなし（All）の場合のみセクションを表示する。

## 変更内容

### タスク 0: 計画ファイル保存 + 作業ブランチ作成

- `docs/plan/v0.8.1.md` に計画を保存
- `fix/hide-repo-costs-on-filter` ブランチを作成

### タスク 1: `repoFilter` に応じて RepositoryCosts の表示を制御

**変更対象**: `src/routes/dashboard.tsx` L65

**受入条件**:
- WHEN `repoFilter === undefined`（All 選択時）THEN `RepositoryCosts` セクションが表示される
- WHEN `repoFilter !== undefined`（特定リポジトリ選択時）THEN `RepositoryCosts` セクションが非表示になる

**実装**: JSX の条件レンダリングを追加

```tsx
{repoFilter === undefined && <RepositoryCosts rows={repoCosts} />}
```

**補足**: `getRepositoryCosts()` クエリは `hasUncategorized`（`RepositoryFilter` の「未分類」オプション表示制御）に必要なため、クエリ自体は引き続き実行する。
